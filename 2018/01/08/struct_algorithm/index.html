单链表逆序问题

  • 单链表逆序.
    如果空间复杂度为O(N), 则可以使用栈做缓存, 重新建立链表即可.
    如果空间复杂度为O(1).
    1 需要注意联表长度需要 >=2.
    2 判断头结点的next为NULL.
    3 交换指针地址.
    4 判断终止条件.

递归问题的思想

  • 递归搞了好些…还是一本算法书上说的有道理
    你只不过再找这样的一个公式 F(n) = F(N-1) + F(N-2)
    还有一个终止条件.
    栈空间压入顺序: F(n)/F(n-1)/F(n-2)/F(n-3)……F(2)/F(1)
    栈出去的顺序: F(1)/F(2)…F(n-3)/F(n-2)/F(n-1)/F(n)

  • 以上程序对栈内存极具毁灭性. 我的8G内存, 10000左右报了栈溢出…
    心塞.

  • 递归调用产生了太多的冗余调用, 这个在python中通过装饰器可以节省计算.
    通过{fibonna(N):vaule}方式.

  • 如果你有兴趣, 减少冗余调用. 可以自己试一试, 反正我是过了…:)

变种问题

  • 判断树结构是否为平衡二叉树. 左子树和右子树的深度差值不超过1
  • 递归思想解决
    “””
    叶子节点的深度值为0.
    返回值为1
    非叶子节点.
    判断左子树和右子树的差, 如果绝对值大于1, 则证明为非平衡2茶树.
    如果绝对值<=1, 择取最大值返回.
    “””
  • 我要在稿纸上先捋一遍看下逻辑是不是通的.
  • 代码逻辑是通的的. 可以搞定.
  • 这个还可以变种为求树的深度.

二叉排序树查找

  • 二叉排序树: 以当前树节点的值为依据, 左小右大.

  • 递归思想.
    “””
    当前节点等于查找的值返回True.
    当前节点的值大于value,进入左子树查找…否则进入右子树查找.
    此种递归走的是单一路径的, 直走从根节点到所在值的节点(叶子节点或非叶子节点).
    “””

  • 稿纸上走了一遍代码…无逻辑错误.:)

折半查找

  • 好坑啊, 本来打算现在写排序的…结果折半还没写. 没关系写了就ok了.

  • 折半前提, 有序数组. 通过不断的去中值缩小范围. 来实现查找. log(2\N)时间复杂度.
    “””
    个人习惯, 我喜欢将数组有序默认为升序.
    下标begin end.
    begin < end 时查找.
    (begin + end)/2 == value 时返回true.
    (begin + end)/2 > value, end = value.
    (begin + end)/2 < value, begin = value.
    begin > end 时返回False.
    “””

  • 写完了…不得不说, 好无聊的折半查找.
